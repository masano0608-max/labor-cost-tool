# ステップ3：GitHub Pages をオンにする

コードのアップロード（ステップ2）が終わったら、サイトを公開する設定をします。

---

## 何をするか

**「GitHub Actions でサイトを表示する」という設定を選ぶ**だけです。

---

## 手順

### ① GitHub のリポジトリのページを開く

- ブラウザで https://github.com を開く
- ログインしていなければログイン
- **自分が作ったリポジトリ**（ステップ1で作ったもの）をクリックして開く

---

### ② Settings（設定）を開く

- リポジトリのページの**一番上**にあるタブが並んでいる
- その中で **「Settings」** をクリック
  - 「Code」「Issues」「Pull requests」の**右側**にあります
  - 歯車のアイコンが目印

---

### ③ 左のメニューから Pages を選ぶ

- Settings の画面で、**左側**にメニューが並んでいる
- 下のほうにある **「Pages」** をクリック
  - 「Code and automation」という見出しの下にあります
  - 見つからない場合は、左メニューを下にスクロールする

---

### ④ Source で GitHub Actions を選ぶ

- 画面の「Build and deployment」というところがある
- その中に **「Source」** という項目がある
- Source の右側に **ドロップダウン（選択できる矢印付きの枠）** がある
- それをクリックして、**「GitHub Actions」** を選ぶ

---

### ⑤ それで完了

- 保存ボタンはないので、選んだ時点で設定完了です
- 2〜3分待つと、ページの上のほうに
  - **「Your site is live at ○○○」** のような表示が出る
- そのURLをクリックすると、サイトが開きます

---

## 画面のイメージ

```
[Code] [Issues] [Pull requests] ... [Settings]  ← ここをクリック
────────────────────────────────────────────

Settings の左メニュー：
  General
  ...
  Code and automation
    Actions
    Pages  ← ここをクリック
    ...

「Build and deployment」のところ：
  Source: [Deploy from a branch ▼]  ← これをクリック
            └ GitHub Actions を選ぶ
```

---

## わからないとき

- **Pages が見つからない** → 左メニューをしっかり下までスクロールする
- **GitHub Actions が出てこない** → Source の横の「▼」をクリックすると選べる
- **サイトが開かない** → 2〜3分待ってからもう一度試す。Actions タブで緑の✓が出ているか確認
