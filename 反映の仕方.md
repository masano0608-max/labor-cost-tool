# 変更をネットに反映する方法

「修正したのにサイトに反映されない」場合の解決手順です。

---

## いまの状態

- パソコンの中には最新のファイルがある
- ネット上（GitHub）には古いまま
- → **アップロード（プッシュ）する必要がある**

---

## やり方（3つから選ぶ）

---

### 方法A：GitHub のサイトからファイルを置き換える（いちばん簡単）

**パソコンに詳しくなくてもできます。**

1. ブラウザでこのアドレスを開く  
   https://github.com/masano0608-max/labor-cost-tool

2. **「2026.2.26」** というフォルダをクリックする

3. 中に入ったら、右上の **「Add file」** をクリックする

4. **「Upload files」** を選ぶ

5. Finder でこのフォルダを開く  
   `パソコン → ユーザー → masanotanaka → 2026.2.26`  
   （または Spotlight で「2026.2.26」と検索）

6. フォルダの中の次のファイルを**すべて**選んで、  
   ブラウザの「Drag files here」のところにドラッグ＆ドロップする  
   - index.html  
   - app.js  
   - calculator.js  
   - test.html  
   - test.js  
   - test-cases.js  
   - calculator.test.js  
   - package.json  
   - styles.css  
   - history.html  
   - README.md  

7. 下の方にある **「Commit changes」** をクリックする

8. **1〜2分待つ**

9. このアドレスでサイトを開いて確認する  
   https://masano0608-max.github.io/labor-cost-tool/2026.2.26/

※ 同じ名前のファイルは上書きされます。新しいファイルは自動で追加されます。

---

### 方法B：GitHub Desktop を使う

**GitHub Desktop を入れている場合**

1. GitHub Desktop を開く

2. 左上の「Current Repository」で「labor-cost-tool」を選ぶ

3. 真ん中あたりに「Push origin」ボタンが出ていたらクリック

4. 出ていなければ、上のメニューから **Repository** → **Push** を選ぶ

5. 1〜2分待ってからサイトを確認

---

### 方法C：ターミナルからプッシュする

**ターミナルを使ったことがある人向け**

1. ターミナルを開く（⌘＋スペース → 「ターミナル」と入力）

2. 次のコマンドをコピーして貼り付け、Enter を押す  
   ```
   cd /Users/masanotanaka/2026.2.26 && git push origin main
   ```

3. ユーザー名やパスワードを聞かれたら入力  
   （パスワードの代わりに Personal Access Token が必要な場合があります）

---

## うまくいかないとき

- **GitHub にログインできていない**  
  → ブラウザで https://github.com を開き、ログインする

- **「Commit changes」を押したのに変わらない**  
  → 2〜3分待ってから、サイトを開き直す（F5 または ⌘＋R で再読み込み）

- **サイトが 404 になる**  
  → 「404エラーの直し方.md」を参照する
