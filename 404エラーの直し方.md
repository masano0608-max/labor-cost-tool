# 404 エラーの直し方

## 原因

GitHub には Static HTML 用の設定（`static.yml`）しか入っておらず、**`index.html` などのファイルがありません**。
一方、あなたのパソコン（ローカル）には正しいファイル一式があります。

---

## 解決方法：強制プッシュする

**ターミナル（Terminal）を開いて**、次のコマンドを **1行ずつ** 実行してください。

```bash
cd /Users/masanotanaka/2026.2.26
git push -u origin main --force
```

- 初回は GitHub のユーザー名とパスワード（またはトークン）を聞かれる場合があります
- 成功すると `index.html` などが GitHub に反映されます

---

## うまくいかない場合

### ログインできない場合

1. **GitHub Desktop** を使っている場合：GitHub Desktop から「Push origin」を実行
2. または **GitHub のウェブサイト**で：
   - リポジトリ → **Add file** → **Upload files**
   - `index.html`、`styles.css`、`app.js`、`history.html`、`test.html` をドラッグ＆ドロップ
   - **Commit changes** をクリック

### プッシュ後にやること

1. 1〜2 分待つ
2. **Actions** タブで「Deploy to GitHub Pages」が緑の✓になるか確認
3. **Settings** → **Pages** の URL にアクセスしてサイトが表示されるか確認
