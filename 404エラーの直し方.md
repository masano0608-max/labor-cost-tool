# 404 エラーの直し方

## 原因

**`index.html` が GitHub にアップロードされていません。**
ローカルにはあるのに、GitHub のリポジトリには入っていない状態です。

---

## 解決方法：もう一度 push する

ターミナルで **次のコマンドを順番に** 実行してください。

### 1. フォルダへ移動

```bash
cd /Users/masanotanaka/2026.2.26
```

### 2. デプロイを実行

```bash
./deploy.sh https://github.com/masano0608-max/labor-cost-tool.git
```

これで `index.html` を含むファイル一式が GitHub に送られます。

---

## うまくいかない場合

`./deploy.sh` が動かない場合は、次のコマンドを 1 行ずつ実行：

```bash
cd /Users/masanotanaka/2026.2.26
git add -A
git commit -m "Add all files for deploy"
git branch -M main
git remote remove origin 2>/dev/null
git remote add origin https://github.com/masano0608-max/labor-cost-tool.git
git push -u origin main
```

※ GitHub にログインしていないと push できません。必要ならユーザー名とパスワード（またはトークン）の入力が求められます。

---

## 完了後

1. 1〜2 分待つ
2. **Actions** タブでワークフローが成功しているか確認
3. **Settings** → **Pages** の URL にアクセスして表示を確認
