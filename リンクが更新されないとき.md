# リンクが更新されないとき

ファイルをアップロードしたのに、サイトの表示が変わらない場合の確認手順です。

---

## 1. 3〜5分待つ

GitHub Pages の反映には **3〜5分** かかることがあります。  
すぐには変わらないので、少し時間を置いてから確認してください。

---

## 2. デプロイが成功しているか確認する

1. https://github.com/masano0608-max/labor-cost-tool を開く

2. 上のタブで **「Actions」** をクリック

3. 左側の **「Deploy to GitHub Pages」** をクリック

4. 一番上にある実行の **丸いアイコン** を確認
   - **緑の ✓** → 成功（あとは待つか、キャッシュを消す）
   - **赤の ✗** → 失敗（下の「うまくいかない場合」を参照）
   - **黄色の ○**（回転中）→ まだ実行中なので数分待つ

---

## 3. ブラウザのキャッシュを消す

古いページが表示されている可能性があります。

### 方法A：ハードリロード（簡単）

- **Mac**: `⌘ + Shift + R`
- **Windows**: `Ctrl + Shift + R`

### 方法B：シークレットウィンドウで開く

- 新しいシークレット／プライベートウィンドウで、サイトの URL を開く

### 方法C：別のブラウザで開く

- 普段使っていないブラウザ（Chrome、Safari など）で同じ URL を開く

---

## 4. 正しい URL で開いているか確認する

次のどちらかの URL を試してください。

```
https://masano0608-max.github.io/labor-cost-tool/2026.2.26/
```

```
https://masano0608-max.github.io/labor-cost-tool/
```

※ どちらか片方だけ動く場合があります。

---

## 5. Pages の設定を確認する

1. https://github.com/masano0608-max/labor-cost-tool を開く

2. **「Settings」** をクリック

3. 左メニューの **「Pages」** をクリック

4. **「Build and deployment」** の **Source** を確認
   - **「GitHub Actions」** になっていれば OK
   - **「Deploy from a branch」** のときは、**「GitHub Actions」** に切り替える

---

## 6. 手動でデプロイを実行する（Actions から）

1. https://github.com/masano0608-max/labor-cost-tool を開く

2. **「Actions」** タブをクリック

3. 左の **「Deploy to GitHub Pages」** をクリック

4. 右側の **「Run workflow」** をクリック

5. **「Run workflow」** ボタン（緑）をもう一度クリック

6. 3〜5分待ってから、サイトを再度確認する

---

## 7. ファイルの置き場所を確認する

アップロードしたファイルが **2026.2.26** フォルダ内にあるか確認してください。

1. https://github.com/masano0608-max/labor-cost-tool を開く

2. **「2026.2.26」** フォルダをクリック

3. `index.html` や `styles.css` が一覧に出ているか確認

※ リポジトリの直下（2026.2.26 の外）に置いてしまうと、正しく表示されないことがあります。
