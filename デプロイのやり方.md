# ステップ2の詳しいやり方

「ターミナルでコマンドを実行する」がわからない方向けの説明です。

---

## ステップ2って何をするの？

**GitHub に、このフォルダのファイルをアップロードする**作業です。

そのために「ターミナル」という画面で、決まった文字を入力します。

---

## ① ターミナルを開く

### Mac の場合

1. キーボードで **Command（⌘）＋ スペース** を押す
2. 「Spotlight」という検索窓が出る
3. **「ターミナル」** と入力
4. **Enter** を押す
5. 黒い画面（または白い画面）が出たらOK

### Windows の場合

1. キーボードで **Windows キー** を押す
2. **「cmd」** と入力
3. **Enter** を押す
4. 黒い画面が出たらOK

---

## ② このフォルダに移動する

開いたターミナルに、次の**1行**をそのままコピーして貼り付け、**Enter** を押します。

```
cd /Users/masanotanaka/2026.2.26
```

※「このフォルダが別の場所にある」場合は、Finder でこのフォルダを右クリック → 「パスをコピー」→ 上記の部分をそのパスに変えます。

---

## ③ デプロイを実行する

次に、次の**1行**をコピーして貼り付け、**Enter** を押します。

**※そのまま実行するとエラーになります。必ず赤字の部分を自分の情報に変えてください。**

```
./deploy.sh https://github.com/あなたのユーザー名/リポジトリ名.git
```

**変えるところ：**
- `あなたのユーザー名` → GitHub のユーザー名（例：tanaka）
- `リポジトリ名` → ステップ1で作ったリポジトリの名前（例：labor-cost-tool）

**例：**
- ユーザー名が `tanaka`、リポジトリ名が `labor-cost-tool` の場合：
```
./deploy.sh https://github.com/tanaka/labor-cost-tool.git
```

---

## ④ ユーザー名とパスワードを聞かれたら

- **Username**：GitHub のユーザー名を入力
- **Password**：GitHub のパスワードは使えません。代わりに「Personal Access Token」が必要な場合があります
  - 作る方法：GitHub → 右上のアイコン → Settings → Developer settings → Personal access tokens → Generate new token
  - 権限で `repo` にチェックを入れて作成し、表示されたトークンを Password の欄に貼り付け

---

## うまくいかないとき

| こんなとき | やること |
|-----------|----------|
| `command not found` と出る | ②の `cd` のコマンドを先に実行したか確認。フォルダの場所が違う可能性も |
| `Permission denied` | ターミナルで `chmod +x deploy.sh` を実行してから、もう一度 ③ を試す |
| `repository not found` | リポジトリのURLが合っているか確認。ステップ1でリポジトリを作ったか確認 |
| パスワードが通らない | Personal Access Token を使う（通常のパスワードは不可） |
